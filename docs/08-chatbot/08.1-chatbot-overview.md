# 08.1 - Visión funcional del chatbot

## Objetivo de producto

El chatbot público debe:

1. orientar al visitante sobre Equipo PUCH y Fundación PROCLADE
2. responder con acciones navegables cuando sea posible
3. registrar dudas no resueltas para que el equipo admin las mejore

## Capacidades actuales

- sesión reutilizable por `sessionId`
- respuestas por HTTP
- canal WebSocket público
- sugerencias
- feedback útil/no útil
- contexto dinámico de noticias, retos, superhéroes y delegaciones
- aviso inicial para no introducir datos sensibles

## Capas del sistema

1. widget frontend
2. orquestador backend
3. motor de matching
4. knowledge base administrable
5. servicios de contexto dinámico
6. panel admin con métricas y tiempo real

## Tipos de respuesta

- `DIRECT_ANSWER`
- `CLARIFICATION`
- `FALLBACK`

## Datos persistidos

- sesiones
- mensajes
- feedback
- preguntas no resueltas
- configuración admin del motor
