# 08.1 - Vision funcional del chatbot

## Objetivo de producto

Proveer un asistente conversacional web que:

1. Ayude al usuario publico a encontrar informacion de la ONG.
2. Responda con datos utiles y acciones navegables (CTA).
3. Permita mejora continua por parte de administradores.

## Arquitectura funcional resumida

1. Frontend widget (`features/chatbot`) envia mensaje con `sessionId` y `pageContext`.
2. Backend orquestador aplica motor de matching (keyword, fuzzy, semantic, context).
3. El resultado se clasifica en `DIRECT_ANSWER`, `CLARIFICATION` o `FALLBACK`.
4. Si procede, se enriquece con contexto dinamico de noticias, retos, superheroes o delegaciones.
5. Admin puede gestionar knowledge/intents/config y revisar metricas/unresolved.

## Contratos clave

Endpoint publico:

- `POST /chatbot/message`

Respuesta (`data`):

- `sessionId`
- `replyType`
- `answer`
- `confidence`
- `detectedIntentCode`
- `suggestions[]`
- `ctaLinks[]`

## KPI funcionales recomendados

- Tasa de `DIRECT_ANSWER` sobre total mensajes BOT.
- Ratio de `FALLBACK`.
- Preguntas no resueltas por periodo (`UnresolvedQuestion`).
- Ratio de feedback positivo (`helpful=true`).
