# 08.5 - Motor de matching y contexto dinámico

## Pipeline actual

1. normalización del mensaje
2. detección temprana de respuestas conversacionales o fuera de dominio
3. recuperación de candidatos desde knowledge base
4. scoring híbrido:
   - keyword
   - fuzzy
   - semantic
   - context
5. aplicación de umbrales
6. enriquecimiento con contexto dinámico
7. safety guard final
8. persistencia de la respuesta

## Configuración

Variables de entorno:

- `CHATBOT_WEIGHT_KEYWORD`
- `CHATBOT_WEIGHT_FUZZY`
- `CHATBOT_WEIGHT_SEMANTIC`
- `CHATBOT_WEIGHT_CONTEXT`
- `CHATBOT_THRESHOLD_DIRECT_ANSWER`
- `CHATBOT_THRESHOLD_CLARIFICATION`
- `CHATBOT_FUZZY_INTERNAL_MIN`

Además, el panel admin puede ajustar pesos y umbrales en runtime.

## Persistencia de trazas útiles

En `ChatMessage.meta` se guarda información útil para análisis:

- `pageContext`
- `suggestions`
- `ctaLinks`
- `scoreBreakdown`
- `candidateId`

## Contexto dinámico

El chatbot puede enriquecer respuestas usando:

- noticias
- retos
- superhéroes
- delegaciones

Esto evita que todo tenga que estar codificado como respuesta estática.
