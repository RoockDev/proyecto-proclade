# 08.5 - HU40 Motor inteligente de matching

## Objetivo

Documentar la implementación de HU-40 sobre la base de HU-39:

- Motor híbrido de scoring explicable.
- Umbrales y pesos configurables por entorno.
- Endpoints de sugerencias y feedback.
- Integración frontend de sugerencias dinámicas y valoración de respuestas.

---

## Pipeline implementado

1. `normalize + tokenize` del mensaje.
2. Recuperación de candidatos desde `KnowledgeItem` + `IntentPhrase`.
3. Cálculo de:
- `keywordScore`
- `fuzzyScore`
- `semanticScore`
- `contextScore`
4. Composición:

```txt
finalScore =
  W_keyword * keywordScore +
  W_fuzzy * fuzzyScore +
  W_semantic * semanticScore +
  W_context * contextScore
```

5. Decisión:
- `finalScore >= directAnswer` -> `DIRECT_ANSWER`
- `clarification <= finalScore < directAnswer` -> `CLARIFICATION`
- `finalScore < clarification` -> `FALLBACK`

6. Persistencia de traza técnica en `ChatMessage.meta`:
- `scoreBreakdown`
- `candidateId`
- `pageContext`

---

## Scores en HU-40

### `keywordScore`

- Coincidencia ponderada de tokens relevantes entre query y candidato.
- Usa pesos mayores para términos críticos de dominio (ej: `donar`, `noticias`, `superheroes`).
- Ignora stopwords en español.

### `fuzzyScore`

- Similaridad aproximada para typos y escritura imperfecta.
- Combina:
  - Jaro-Winkler de frase completa.
  - Levenshtein normalizado por tokens.
- Se aplica penalización para queries demasiado cortas y un umbral interno mínimo.

### `semanticScore`

- Implementado con `TF-IDF + cosine similarity`.
- Se cachean artefactos semánticos por fingerprint de candidatos para reducir coste.

### `contextScore`

- Composición:
  - recencia de intención detectada en sesión,
  - match de `pageContext` con `route` del candidato,
  - continuidad temporal de conversación.

## Configuración por entorno (sin recompilar)

Variables añadidas en `backend/.env.example`:

```env
CHATBOT_WEIGHT_KEYWORD=0.35
CHATBOT_WEIGHT_FUZZY=0.25
CHATBOT_WEIGHT_SEMANTIC=0.30
CHATBOT_WEIGHT_CONTEXT=0.10
CHATBOT_THRESHOLD_DIRECT_ANSWER=0.75
CHATBOT_THRESHOLD_CLARIFICATION=0.50
CHATBOT_FUZZY_INTERNAL_MIN=0.25
```

Servicio de configuración:
- `backend/src/chatbot/chatbot-matching-config.service.ts`

Servicio motor:
- `backend/src/chatbot/chatbot-matching-engine.service.ts`

## Endpoints HU-40 implementados

### `GET /chatbot/suggestions`

Query params:
- `sessionId?`
- `pageContext?`
- `limit?` (1..8)

Salida:
- quick replies contextuales para el widget.

### `POST /chatbot/feedback`

Body:

```json
{
  "sessionId": "uuid",
  "messageId": 123,
  "helpful": true
}
```

Comportamiento:
- valida sesión abierta y mensaje BOT.
- guarda feedback en `ChatMessage.meta.feedback`.

---

## Frontend HU-40

Cambios principales:

- `useChatbot`:
  - carga sugerencias dinámicas al abrir panel,
  - envía feedback por mensaje.
- `ChatbotWidget`:
  - dispara `loadSuggestions(pageContext)` al abrir.
- `ChatbotMessageItem`:
  - botones de valoración (thumbs up/down) por mensaje del bot con `messageId`.
