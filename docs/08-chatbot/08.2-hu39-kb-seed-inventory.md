# 08.2 - Inventario funcional de conocimiento base

## Propósito actual del conocimiento base

La base inicial del chatbot ya no debe entenderse como una seed histórica aislada, sino como el conjunto mínimo de conocimiento administrable con el que el sistema arranca y que luego el panel admin puede evolucionar.

## Fuentes funcionales que alimentan respuestas

### Base estable

- `ChatbotIntent`
- `ChatbotIntentPhrase`
- `KnowledgeItem`
- `backend/src/chatbot/seed/chatbot-seed.source.ts`

### Contexto vivo

- noticias
- retos
- superhéroes
- delegaciones

## Principio vigente

El chatbot no debe depender de documentos externos en runtime. Todo el conocimiento operativo debe vivir en:

1. seed/bootstrap de chatbot
2. tablas del módulo chatbot
3. datos vivos de dominio

## Consecuencia para mantenimiento

Si cambia contenido público importante y se quiere que el chatbot lo explique mejor, hay dos caminos:

1. ajustar knowledge items/intents desde admin
2. ampliar la lógica de contexto dinámico si el caso depende de datos vivos
