# 08.7 - Admin chatbot: gobernanza y metricas

## Objetivo

Documentar la capa admin del chatbot implementada en `/admin/chatbot` para que el equipo mantenga calidad conversacional sin tocar codigo fuente.

## Endpoints admin disponibles

Base: `/admin/chatbot` (JWT + rol `ADMIN`)

### Metricas y seguimiento

- `GET /metrics`
- `GET /unresolved`
- `PATCH /unresolved/:id/resolve`
- `PATCH /unresolved/:id/reopen`
- `DELETE /unresolved/:id`

### Knowledge base

- `GET /knowledge-items`
- `POST /knowledge-items`
- `PATCH /knowledge-items/:id`
- `DELETE /knowledge-items/:id`

### Intents y frases

- `GET /intents`
- `POST /intents`
- `PATCH /intents/:id`
- `POST /intents/:intentId/phrases`
- `PATCH /intents/:intentId/phrases/:phraseId`

### Configuracion de motor

- `PATCH /config`

Permite actualizar en runtime:

- pesos (`keyword`, `fuzzy`, `semantic`, `context`)
- umbrales (`directAnswer`, `clarification`)
- `fuzzyInternalMin`

## Flujo operativo recomendado

1. Revisar metricas y unresolved semanalmente.
2. Agrupar preguntas repetidas por intencion.
3. Actualizar knowledge/intents/phrases.
4. Ajustar pesos/umbrales en cambios controlados.
5. Volver a medir impacto en ratio direct/fallback.

## Buenas practicas

1. Evitar respuestas excesivamente largas o ambiguas.
2. Mantener CTA accionables y coherentes con rutas reales.
3. No introducir datos sensibles ni promesas no soportadas por negocio.
4. Versionar cambios relevantes de tuning en PR y docs.
