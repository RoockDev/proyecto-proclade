# 08.3 - Persistencia y bootstrap del conocimiento

## Estado actual

El chatbot arranca con persistencia real en PostgreSQL y ya no depende de lanzar un `prisma:seed` manual desde operación diaria.

## Modelos implicados

- `ChatbotIntent`
- `ChatbotIntentPhrase`
- `KnowledgeItem`
- `ChatSession`
- `ChatMessage`
- `UnresolvedQuestion`
- `ChatbotAdminConfig`

## Bootstrap actual

Archivo:

- `backend/prisma/bootstrap-system.ts`

Secuencia:

1. asegurar roles base
2. asegurar admin inicial
3. ejecutar `seedChatbotKnowledge(prisma)`

## Fuente base de conocimiento

Archivo principal:

- `backend/src/chatbot/seed/chatbot-seed.source.ts`

Semilla efectiva:

- `backend/prisma/seeds/chatbot-knowledge.seed.ts`

## Conclusión operativa

El conocimiento inicial del chatbot forma parte del arranque del sistema y debe considerarse bootstrap de plataforma, no seed opcional de demo.
