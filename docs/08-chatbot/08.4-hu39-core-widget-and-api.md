# 08.4 - Widget, API pública y sesión

## Frontend

Componente principal:

- `frontend/src/features/chatbot/components/ChatbotWidget/ChatbotWidget.tsx`

Comportamiento actual:

- vive dentro de `PublicLayout`
- construye `pageContext` desde la URL
- carga sugerencias al abrir
- permite feedback
- navega por CTA internas o externas
- muestra aviso de privacidad hasta la primera interacción

## API HTTP pública

- `POST /chatbot/sessions`
- `POST /chatbot/message`
- `GET /chatbot/suggestions`
- `POST /chatbot/feedback`
- `GET /chatbot/health`

## WebSocket público

Namespace:

- `/chatbot`

Eventos entrantes:

- `initSession`
- `sendMessage`
- `getSuggestions`
- `registerFeedback`
- `getHealth`

Eventos salientes:

- `sessionInitialized`
- `messageReceived`
- `suggestionsReceived`
- `feedbackRegistered`
- `healthStatus`
- `error`

## Sesión

El orquestador puede reutilizar una sesión existente o crear una nueva. La conversación queda persistida en `ChatSession` y `ChatMessage`.
