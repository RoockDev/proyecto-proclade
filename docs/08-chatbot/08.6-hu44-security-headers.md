# 08.6 - Cabeceras y superficie de seguridad relacionada

## Qué cubre este documento

Aunque las cabeceras se aplican a toda la plataforma, aquí interesan especialmente por el chatbot y sus canales HTTP/WebSocket.

## Cabeceras actuales en Nginx

- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: SAMEORIGIN`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Permissions-Policy: geolocation=(), microphone=(), camera=(), gyroscope=(), magnetometer=(), accelerometer=(), payment=(), interest-cohort=()`
- `Content-Security-Policy`

## Relación con chatbot y servicios externos

- `connect-src 'self'` cubre API y Socket.IO sobre el mismo host.
- `frame-src` permite Google Maps, pero el frontend decide funcionalmente si mostrarlo según preferencias de cookies.
- `style-src` y `font-src` siguen permitiendo Google Fonts en el estado actual.

## Qué revisar al tocar chatbot

1. si se añade un nuevo origen externo, revisar CSP
2. si se cambia el path de Socket.IO, revisar `nginx/default.conf`
3. si se endurece CSP, validar widget, admin realtime y mapas
