# 03.5 - Operacion Backend

## Arranque en desarrollo (Docker)

Desde raiz:

```bash
docker compose up --build
```

## Comandos utiles backend

Ejecutar dentro de contenedor backend:

```bash
docker exec -it proyectoproclade-backend-1 sh
```

Comandos internos:

```bash
yarn prisma generate
yarn prisma migrate deploy
yarn prisma:seed
yarn test
yarn test:e2e
yarn build
```

## Migraciones

- Fuente: `backend/prisma/migrations/`
- En Docker Compose se ejecuta `prisma migrate deploy` en arranque backend.

## Variables de entorno backend

Ver: `backend/.env.example`

Criticas:

- `DATABASE_URL`
- `JWT_SECRET`
- `JWT_EXPIRES_IN`
- `GOOGLE_CLIENT_ID`
- `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`, `MAIL_FROM`
- `FRONTEND_URL`
- `RESET_PASSWORD_TTL_MINUTES`

## Checklist de soporte backend

1. API responde en `GET /news`.
2. DB conectada y migraciones aplicadas.
3. Login emite token valido.
4. Endpoints admin rechazados sin JWT/rol.
5. Correo de reset sale correctamente en entorno configurado.
