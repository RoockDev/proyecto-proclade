# 03.5 - Operación backend

## Arranque recomendado

Desde la raíz del proyecto:

```bash
docker compose up --build
```

## Secuencia real del backend en Docker

Archivo:

- `backend/scripts/docker-start-dev.sh`

Secuencia:

```bash
yarn exec prisma generate
yarn exec prisma migrate deploy
yarn prisma:init-system
exec yarn start
```

## Comandos útiles

Entrar al contenedor:

```bash
docker compose exec backend sh
```

Comandos internos habituales:

```bash
yarn build
yarn test --runInBand
yarn prisma:generate
yarn prisma:migrate:deploy
yarn prisma:init-system
```

## Bootstrap del sistema

Archivo:

- `backend/prisma/bootstrap-system.ts`

Funciones:

1. asegurar rol `ADMIN`
2. asegurar rol `USER`
3. crear o verificar admin inicial
4. cargar base del chatbot

## Uploads

Rutas locales:

- `backend/uploads/news`
- `backend/uploads/superheroes`
- `backend/uploads/human-books`

Servicio estático:

- el backend expone `/uploads/*` desde `main.ts`

## Operaciones de reset

### Parar sin perder datos

```bash
docker compose down
```

### Reset completo

```bash
docker compose down -v
```

Efecto:

- borra la base de datos del volumen Docker
- obliga a reejecutar bootstrap en el siguiente arranque

Importante:

- no limpia necesariamente los ficheros locales de `backend/uploads` porque es un bind mount

## Incidencias backend frecuentes

### El backend no arranca

Revisar:

- `JWT_SECRET`
- `DATABASE_URL`
- logs del contenedor

### El admin inicial no entra

Posibles causas:

- `SYSTEM_ADMIN_PASSWORD` inválida
- el admin ya existía y se cambió el `.env` después
- la base de datos no se reseteó antes del primer arranque real

### Colabora devuelve error

Revisar:

- `SMTP_HOST`
- `SMTP_USER`
- `SMTP_PASS`
- `CONTACT_FORM_TO`

Si no hay SMTP configurado:

- `forgot-password` no podrá enviar email real
- `colabora/contact` devolverá `503`
