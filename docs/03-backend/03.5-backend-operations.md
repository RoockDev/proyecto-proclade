# 03.5 - Operacion Backend

## Arranque recomendado (Docker)

Desde raiz:

```bash
docker compose up --build
```

## Comandos utiles backend

Entrar al contenedor backend:

```bash
docker exec -it proyectoproclade-backend-1 sh
```

Comandos internos:

```bash
yarn prisma:generate
yarn prisma:migrate:deploy
yarn prisma:seed
yarn lint
yarn build
yarn test
yarn test:e2e
```

## Migraciones

- Fuente: `backend/prisma/migrations/`
- En Docker Compose se ejecuta `prisma migrate deploy` al arrancar backend.

## Uploads

Rutas locales:

- `backend/uploads/news`
- `backend/uploads/superheroes`
- `backend/uploads/human-books`

Puntos clave:

1. `docker-compose.yml` monta `./backend/uploads:/app/uploads`.
2. Si se borra el volumen o carpeta local, se pierden ficheros no versionados.

## Variables de entorno backend

Referencia: `backend/.env.example`

Criticas:
- `DATABASE_URL`
- `JWT_SECRET`
- `JWT_EXPIRES_IN`
- `CORS_ORIGIN`
- `GOOGLE_CLIENT_ID`
- `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`, `MAIL_FROM`
- `FRONTEND_URL`, `RESET_PASSWORD_TTL_MINUTES`
- `CHATBOT_WEIGHT_*`, `CHATBOT_THRESHOLD_*`, `CHATBOT_FUZZY_INTERNAL_MIN`

## Checklist rapido de soporte backend

1. API responde en `GET /news` y `GET /regions`.
2. DB conectada y migraciones aplicadas.
3. Login emite token valido.
4. Endpoints admin rechazan acceso sin JWT/rol.
5. Chatbot responde en `GET /chatbot/health`.
6. Upload y descarga de PDF/imagenes funciona en rutas `/uploads/*`.
