# 03.3 - Modelo de Datos

Fuente de verdad: `backend/prisma/schema.prisma`.

## Entidades principales

### User

Campos relevantes:

- `id` (PK)
- `email` (unique)
- `passwordHash`
- `name`, `surname`
- `google` (boolean)
- `resetPasswordTokenHash`, `resetPasswordExpiresAt`
- `deletedAt` (soft delete)
- `createdAt`, `updatedAt`

Relaciones:

- N:M con `Role` (relacion `UserRoles`)
- 1:N con `News` (`createdNews`)

### Role

Campos:

- `id` (PK)
- `name` (unique)
- `createdAt`, `updatedAt`

Relaciones:

- N:M con `User`

### News

Campos:

- `id` (PK)
- `title`
- `slug` (unique)
- `excerpt`
- `content` (Text)
- `imageUrl` (nullable)
- `status` (`DRAFT` | `PUBLISHED`)
- `publishedAt` (nullable)
- `createdById` (FK User)
- `deletedAt` (soft delete)
- `createdAt`, `updatedAt`

Relaciones:

- N:1 con `User` (`createdBy`)

## Reglas de negocio relevantes

1. Una noticia solo es visible publicamente si:
- `status = PUBLISHED`
- `deletedAt IS NULL`

2. Publicacion:
- Al crear en `PUBLISHED`, `publishedAt` se asigna automaticamente.
- En update, solo se asigna `publishedAt` la primera vez que pasa a publicado.

3. Slug:
- Se genera siempre en backend.
- Es unico y usa sufijos incrementales (`-2`, `-3`, ...).

4. Borrado logico:
- Usuarios y noticias no se eliminan fisicamente en operaciones funcionales.
