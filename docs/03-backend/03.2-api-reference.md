# 03.2 - API reference

## Base URL

- Backend directo: `http://localhost:3000`
- Vía Nginx: `http://localhost/api`

La referencia funcional recomendada para QA y handover es la vía Nginx.

## Convenciones

1. Todas las respuestas HTTP siguen el contrato:

```json
{
  "success": true,
  "message": "Texto descriptivo",
  "data": {}
}
```

2. Los errores usan el mismo envoltorio con `success: false`.
3. Los endpoints `/admin/*` requieren JWT válido y rol `ADMIN`.
4. `users/me` y `auth/change-password` requieren JWT válido.

## Auth (`/auth`)

- `POST /auth/login`
- `POST /auth/register`
- `POST /auth/forgot-password`
- `POST /auth/reset-password`
- `PATCH /auth/change-password`

Reglas importantes:

- `register` exige `privacyAccepted = true`
- la contraseña debe cumplir la política centralizada
- `forgot-password` no revela si el email existe
