# 03.2 - API Reference

## Base URL

- Backend directo local: `http://localhost:3000`
- Via Nginx: `http://localhost/api`

## Convenciones

1. Respuesta estandar `ApiResponse<T>`.
2. Endpoints `/admin/*` requieren JWT Bearer y rol `ADMIN`.
3. Los endpoints publicos nunca devuelven `passwordHash` ni secretos.

## Auth (`/auth`)

- `POST /auth/login`
- `POST /auth/register`
- `POST /auth/google`
- `POST /auth/forgot-password`
- `POST /auth/reset-password`
- `PATCH /auth/change-password` (JWT)

## Users admin (`/users`) [ADMIN]

- `POST /users`
- `GET /users?includeDeleted=true|false`
- `GET /users/:id`
- `PATCH /users/:id`
- `DELETE /users/:id` (soft delete)
- `POST /users/:id/restore`
- `PATCH /users/:id/real-hero`

## Users perfil (`/users/me`) [JWT]

- `GET /users/me`
- `PATCH /users/me`

## News publica (`/news`)

- `GET /news?limit=1..50`
- `GET /news/:slug`

## News admin (`/admin/news`) [ADMIN]

- `GET /admin/news`
- `POST /admin/news`
- `POST /admin/news/upload-image`
- `PATCH /admin/news/:id`
- `DELETE /admin/news/:id`

## Superheroes publicos (`/superheroes`)

- `GET /superheroes?page=&pageSize=&status=&search=`
- `GET /superheroes/:slug`

## Superheroes admin (`/admin/superheroes`) [ADMIN]

- `GET /admin/superheroes`

- `PATCH /admin/superheroes/:id` (multipart image opcional)

- `PATCH /admin/superheroes/:id/restore`
- `DELETE /admin/superheroes/:id` (soft delete)
- `DELETE /admin/superheroes/:id/permanent`

{
  "success": true,
  "message": "Operacion realizada con exito",

    "key": "value"
  }
}
```

{
  "success": false,
  "message": "Credenciales invalidas",
  "data": null
}
```
