# 03.2 - API Reference

## Base URL

- En backend directo (local): `http://localhost:3000`
- A traves de Nginx: `http://localhost/api`

## Convenciones

1. Todas las respuestas usan `ApiResponse<T>`.
2. Endpoints admin requieren JWT Bearer y rol `ADMIN`.
3. Errores validacion se devuelven con `success: false` y `data: null`.

## Auth (`/auth`)

### POST `/auth/login`

- Auth: publica
- Body:
  - `email: string`
  - `password: string`
- Data response:
  - `accessToken: string`
  - `user: { id, email, name, surname, roles[] }`

### POST `/auth/register`

- Auth: publica
- Body:
  - `name: string`
  - `surname: string`
  - `email: string`
  - `password: string`
- Data response: igual que login

### POST `/auth/google`

- Auth: publica
- Body:
  - `idToken: string`
- Data response: igual que login

### POST `/auth/forgot-password`

- Auth: publica
- Body:
  - `email: string`
- Data response: `null`
- Nota: respuesta generica por seguridad (no expone existencia de email)

### POST `/auth/reset-password`

- Auth: publica
- Body:
  - `token: string`
  - `newPassword: string`
- Data response: `null`

## Users (`/users`) - ADMIN

### POST `/users`

- Auth: `ADMIN`
- Body:
  - `email, name, surname, password`
  - `roles?: RoleName[]`
- Data response: usuario sin `passwordHash`

### GET `/users`

- Auth: `ADMIN`
- Data response: lista de usuarios activos

### GET `/users/:id`

- Auth: `ADMIN`
- Data response: detalle de usuario

### PATCH `/users/:id`

- Auth: `ADMIN`
- Body: campos parciales de usuario
- Data response: usuario actualizado

### DELETE `/users/:id`

- Auth: `ADMIN`
- Data response: `null`
- Nota: aplica soft delete (`deletedAt`)

## News public (`/news`)

### GET `/news?limit=<1..50>`

- Auth: publica
- Query opcional:
  - `limit: number` (1..50)
- Data response: lista de noticias `PUBLISHED`, ordenadas por `publishedAt desc`

### GET `/news/:slug`

- Auth: publica
- Data response: detalle de noticia publicada por slug

## News admin (`/admin/news`) - ADMIN

### GET `/admin/news`

- Auth: `ADMIN`
- Data response: lista de noticias no borradas (draft + published)

### POST `/admin/news`

- Auth: `ADMIN`
- Body:
  - `title`
  - `excerpt` (40..280)
  - `content`
  - `imageUrl?`
  - `status?` (`DRAFT` | `PUBLISHED`)
- Data response: `{ news: News }`

### PATCH `/admin/news/:id`

- Auth: `ADMIN`
- Body: parcial de `CreateNewsDto`
- Regla: si estado actual es `DRAFT` y cambia `title`, se recalcula slug
- Data response: `{ news: News }`

### DELETE `/admin/news/:id`

- Auth: `ADMIN`
- Data response: `null`
- Nota: aplica soft delete

## Ejemplo de respuesta correcta

```json
{
  "success": true,
  "message": "Operacion realizada con exito",
  "data": {
    "key": "value"
  }
}
```

## Ejemplo de error

```json
{
  "success": false,
  "message": "Credenciales invalidas",
  "data": null
}
```
