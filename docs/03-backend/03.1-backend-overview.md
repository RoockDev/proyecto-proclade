# 03.1 - Visión backend

## Stack

- NestJS 11
- Prisma 7
- PostgreSQL
- JWT Bearer
- class-validator / class-transformer
- Socket.IO
- Nodemailer

## Módulos cargados en `AppModule`

- `AuthModule`
- `PrismaModule`
- `UsersModule`
- `NewsModule`
- `ChatbotModule`
- `ChatbotAdminModule`
- `RegionsModule`
- `SuperheroesModule`
- `HumanBooksModule`
- `ChallengesModule`
- `ColaboraModule`

## Capacidades principales por módulo

### `AuthModule`

- Login con email/contraseña
- Registro con `privacyAccepted`
- Forgot/reset password
- Cambio de contraseña autenticado
- Generación de JWT con roles

### `UsersModule`

- CRUD admin de usuarios
- Perfil autenticado `users/me`
- Rol por defecto `USER` con `connectOrCreate`
- Sanitización de respuestas para no exponer hashes ni tokens de reset
- Conversión de usuario a `real hero`

### `NewsModule`

- Noticias públicas publicadas
- CRUD admin
- Upload y descarte de imagen
- Borrado lógico en entidad y limpieza de fichero cuando aplica

### `SuperheroesModule`

- Catálogo público por slug
- CRUD admin
- Gestión de imagen con almacenamiento controlado
- Soft delete, restore y borrado permanente
- Limpieza de imagen huérfana al fallar o borrar

### `RegionsModule`

- Delegaciones públicas activas
- CRUD admin
- Campo de teléfono editable
- Endpoint de opciones para selects admin

### `HumanBooksModule`

- Listado público de libros publicados
- Descarga de PDF por slug
- CRUD admin con PDF local
- Eliminación del PDF al borrar el libro

### `ChallengesModule`

- Listado público de retos activos
- CRUD admin
- Actualización controlada de importes

### `ColaboraModule`

- Validación del formulario de contacto
- Requiere `privacyAccepted`
- Devuelve error real si SMTP no está configurado
- Incluye metadatos RGPD en el correo

### `ChatbotModule`

- Sesiones
- Mensajería pública
- Sugerencias
- Feedback
- Healthcheck
- Matching híbrido
- Persistencia de mensajes y unresolved
- Canal WebSocket público

### `ChatbotAdminModule`

- Métricas
- Preguntas no resueltas
- Knowledge items
- Intents y phrases
- Configuración runtime
- Gateway admin en tiempo real

## Componentes transversales

- `ValidationPipe` global
- `TransformInterceptor`
- `GlobalExceptionFilter`
- `JwtAuthGuard`
- `RolesGuard`
- `MailService`

## Arranque del sistema

El backend no depende de seeds manuales para quedar operativo. En Docker ejecuta:

1. `prisma generate`
2. `prisma migrate deploy`
3. `prisma:init-system`
4. `yarn start`

El bootstrap del sistema:

- asegura roles `ADMIN` y `USER`
- crea o verifica el admin inicial
- carga la base inicial del chatbot
