# 03.1 - Vision Backend

## Stack

- NestJS 11
- TypeScript
- Prisma 7
- PostgreSQL
- JWT + Roles
- class-validator / class-transformer
- Nodemailer

## Modulos activos

1. `AuthModule`
- Login, register, Google Sign-In.
- Forgot/reset password.
- Change password autenticado.

2. `UsersModule`
- CRUD admin de usuarios.
- Perfil autenticado (`/users/me`).
- Gestion de real hero por usuario.

3. `NewsModule`
- Lectura publica de noticias publicadas.
- Gestion admin con upload de imagen y soft delete.

4. `SuperheroesModule`
- Catalogo publico por slug.
- CRUD admin con upload de imagen.
- Soft delete, restore y borrado permanente controlado.

5. `RegionsModule`
- Listado publico de delegaciones activas.
- CRUD admin de delegaciones.

6. `HumanBooksModule`
- Listado publico de libros humanos.
- Descarga de PDF por slug.
- CRUD admin con upload PDF local.

7. `ChallengesModule`
- Gestion admin de retos, incluido update de cantidad recaudada.

8. `ChatbotModule`
- Sesiones de chat, envio de mensaje, sugerencias, feedback, health.
- Motor hibrido de matching + contexto dinamico.

9. `ChatbotAdminModule`
- Gobernanza del chatbot: metricas, unresolved, knowledge base, intents, frases y config de matching.

10. `PrismaModule`
- Acceso a datos centralizado.

## Bootstrap global (`backend/src/main.ts`)

- Creacion de carpeta `uploads` si no existe.
- Publicacion de estaticos en `/uploads`.
- CORS configurable por `CORS_ORIGIN`.
- `ValidationPipe` global (`whitelist`, `forbidNonWhitelisted`, `transform`).
- `TransformInterceptor` global (contrato unificado).
- `GlobalExceptionFilter` global (errores unificados).

## Contrato de salida backend

```ts
{
  success: boolean;
  message: string;
  data: T | null;
}
```

Este contrato aplica tanto en exito como en error.
