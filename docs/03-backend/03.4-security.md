# 03.4 - Seguridad

## Controles implementados

1. Validacion de entrada
- `ValidationPipe` global con `whitelist`, `forbidNonWhitelisted`, `transform`.
- DTOs tipados en todos los modulos de negocio.

2. Autenticacion
- JWT Bearer token.
- `JwtAuthGuard` valida token para rutas protegidas.

3. Autorizacion
- `RolesGuard` + decorador `@Roles`.
- Endpoints admin restringidos a rol `ADMIN`.

4. Credenciales y secretos
- Passwords con `bcrypt`.
- Reset token almacenado hasheado (`sha256`) y con expiracion.

5. Endurecimiento HTTP (Nginx)
- `X-Content-Type-Options`, `X-Frame-Options`, `Referrer-Policy`, `Permissions-Policy`, `Content-Security-Policy`.
- Bloqueo de rutas sensibles (`.env`, `.git`, ficheros ocultos, backups).

6. Control de upload
- Validacion de tipo/tamano para imagenes y PDF.
- Ficheros servidos desde `/uploads` bajo ruta controlada.

7. Respuesta de seguridad
- Error handling unificado con `GlobalExceptionFilter`.
- `forgot-password` devuelve mensaje generico para evitar enumeracion de usuarios.

## Riesgos actuales

1. `CORS_ORIGIN` vacio/mal configurado en entornos reales.
2. Falta rate limiting explicito para login/chatbot/admin.
3. Persistencia local de uploads (sin offsite backup ni antivirus).
4. Deuda de lint en backend que dificulta hardening continuo.

## Recomendaciones para produccion

1. Forzar `JWT_SECRET` robusto y rotacion programada.
2. Definir `CORS_ORIGIN` cerrado por entorno.
3. Incorporar throttling en endpoints sensibles (`/auth/*`, `/chatbot/*`, `/admin/*`).
4. Migrar uploads a storage gestionado (S3 compatible) con politicas de acceso.
5. Activar TLS completo y monitorizar cabeceras de seguridad en cada deploy.
