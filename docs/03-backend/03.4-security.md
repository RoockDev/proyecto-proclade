# 03.4 - Seguridad

## Controles implementados

1. Validacion de entrada
- `ValidationPipe` global.
- DTOs con `class-validator` en auth/users/news.

2. Autenticacion
- JWT Bearer token.
- `JwtAuthGuard` valida firma y expiracion.

3. Autorizacion
- `RolesGuard` + decorador `@Roles`.
- Endpoints de administracion restringidos a `ADMIN`.

4. Hash de credenciales
- Passwords con `bcrypt`.
- Tokens de reset guardados hasheados (`sha256`).

5. Respuestas de seguridad
- `forgot-password` no revela si email existe.

6. Manejo de errores
- `GlobalExceptionFilter` evita payloads inconsistentes.

## Riesgos actuales

1. `JwtAuthGuard` usa fallback de secreto por defecto (`secretKey`) si falta env.
- Recomendacion: eliminar fallback y forzar `JWT_SECRET` obligatorio.

2. No hay rate limiting en login/forgot-password.
- Recomendacion: incorporar throttling por IP y usuario.

3. CORS por env, pero sin whitelist versionada en documentacion operativa.

4. No hay auditoria persistente de acciones admin.

## Recomendaciones para entorno productivo

1. `JWT_SECRET` robusto y rotacion controlada.
2. SMTP con cuenta tecnica dedicada y password de aplicacion.
3. Forzar HTTPS extremo a extremo en servidor final.
4. Integrar logs estructurados y alertas de seguridad.
