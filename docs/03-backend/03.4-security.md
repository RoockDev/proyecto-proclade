# 03.4 - Seguridad

## Controles implementados

### Validación de entrada

- `ValidationPipe` global con:
  - `whitelist`
  - `forbidNonWhitelisted`
  - `transform`
  - `stopAtFirstError`
- DTOs específicos por dominio

### Autenticación y autorización

- JWT Bearer
- `JwtAuthGuard`
- `RolesGuard`
- decorador `@Roles`

### Secretos y credenciales

- `JWT_SECRET` es obligatorio para arrancar
- contraseñas con `bcrypt`
- tokens de recuperación almacenados hasheados
- política de contraseña centralizada:
  - mínimo 8 caracteres
  - 1 mayúscula
  - 1 minúscula
  - 1 número

### Privacidad y protección de datos

- registro exige `privacyAccepted`
- Colabora exige `privacyAccepted`
- `User.privacyAcceptedAt` registra consentimiento en alta
- los correos de Colabora incluyen metadatos RGPD

### Respuesta y superficie HTTP

- `TransformInterceptor` unifica éxito
- `GlobalExceptionFilter` unifica error
- Nginx aplica:
  - `X-Content-Type-Options`
  - `X-Frame-Options`
  - `Referrer-Policy`
  - `Permissions-Policy`
  - `Content-Security-Policy`
- Nginx bloquea `.env`, `.git`, archivos ocultos y backups comunes

### Uploads

- validación de tipo y tamaño
- almacenamiento local bajo `/uploads`
- limpieza de archivos huérfanos en varios flujos

### Sanitización de salidas

Las respuestas de usuario no devuelven:

- `passwordHash`
- `resetPasswordTokenHash`
- `resetPasswordExpiresAt`

## Riesgos actuales

1. El namespace WebSocket `/admin/chatbot` no aplica autenticación ni rol a nivel gateway.
2. No hay rate limiting explícito en `auth`, `chatbot` o panel admin.
3. Los uploads siguen en filesystem local sin backup ni antivirus.
4. `style-src` y `font-src` en CSP siguen permitiendo Google Fonts aunque el consentimiento funcional fuerte se aplica sobre mapas/servicios externos.

## Recomendaciones para siguiente fase

1. Proteger el gateway admin del chatbot con autenticación real.
2. Añadir rate limit en autenticación, chatbot y formularios públicos.
3. Definir backup o storage externo para `uploads`.
4. Revisar CSP y fuentes externas si se quiere una política de privacidad aún más estricta.
