# 02.5 - Decisiones Arquitectonicas Actuales

## ADR-A001 - Contrato de respuesta unificado

- Decision: toda respuesta backend se normaliza a `{ success, message, data }`.
- Estado: vigente.

## ADR-A002 - Seguridad por JWT + Roles

- Decision: endpoints admin protegidos por `JwtAuthGuard` + `RolesGuard`.
- Estado: vigente.

## ADR-A003 - Soft delete por defecto en contenido gestionable

- Decision: borrado logico para usuarios, noticias, regiones, retos y libros humanos.
- Excepcion: superheroes permite purge controlado (`/permanent`) tras desactivacion.

## ADR-A004 - Storage local para assets gestionables

- Decision: imagenes/PDFs gestionados por admin se almacenan en `backend/uploads`.
- Motivo: simplicidad operativa en entorno academico y staging temprano.

## ADR-A005 - Chatbot hibrido (knowledge + contexto dinamico)

- Decision: el chatbot combina knowledge base administrable + consultas dinamicas a tablas reales.
- Motivo: evitar respuestas obsoletas y permitir gobernanza por admin.

## ADR-A006 - Seccion publica Bibliotecas Humanas sin mocks

- Decision: consumir delegaciones y libros desde API real (`/regions`, `/human-books`).
- Motivo: coherencia con modelo real y mantenimiento centralizado en panel admin.

## ADR-A007 - Guard de sesion en frontend por expiracion JWT

- Decision: validar `exp` en cliente y limpiar sesion en `401` via interceptor global Axios.
- Motivo: evitar estado de sesion fantasma y mejorar UX de seguridad.

## Deuda tecnica abierta relevante

1. Pipeline staging aun con placeholders para lint/build/test en deploy workflow.
2. Lint backend con deuda heredada (no bloqueante de estas HUs, pero critica para hardening).
3. Falta observabilidad centralizada (metricas, alertas, trazas estructuradas).
