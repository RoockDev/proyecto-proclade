# 02.3 - C4 Component (Backend)

## Objetivo

Documentar los componentes principales del contenedor backend NestJS.

```mermaid
flowchart LR
  fe["Frontend React"]
  db[("PostgreSQL")]
  smtp["SMTP"]
  google["Google Identity"]

  subgraph api["NestJS API"]
    auth["AuthModule"]
    users["UsersModule"]
    news["NewsModule"]
    superheroes["SuperheroesModule"]
    regions["RegionsModule"]
    humanBooks["HumanBooksModule"]
    challenges["ChallengesModule"]
    chatbot["ChatbotModule"]
    chatbotAdmin["ChatbotAdminModule"]
    prisma["PrismaModule"]

    jwt["JwtAuthGuard"]
    roles["RolesGuard"]
    interceptor["TransformInterceptor"]
    filter["GlobalExceptionFilter"]
  end

  fe --> auth
  fe --> users
  fe --> news
  fe --> superheroes
  fe --> regions
  fe --> humanBooks
  fe --> challenges
  fe --> chatbot
  fe --> chatbotAdmin

  auth --> prisma
  users --> prisma
  news --> prisma
  superheroes --> prisma
  regions --> prisma
  humanBooks --> prisma
  challenges --> prisma
  chatbot --> prisma
  chatbotAdmin --> prisma

  prisma --> db
  auth --> google
  auth --> smtp

  chatbotAdmin --> jwt
  chatbotAdmin --> roles
  users --> jwt
  users --> roles

  auth --> interceptor
  news --> interceptor
  chatbot --> interceptor
  auth --> filter
  news --> filter
  chatbot --> filter
```

## Nota

`TransformInterceptor` y `GlobalExceptionFilter` garantizan el contrato `{ success, message, data }` en todo el backend.
