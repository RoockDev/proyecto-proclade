# 02.2 - C4 Container

## Objetivo

Describir los contenedores y procesos principales que forman el sistema en local.

```mermaid
flowchart LR
  browser["Navegador"]
  smtp["SMTP"]
  maps["Google Maps"]

  subgraph system["Proyecto Proclade"]
    nginx["Nginx :80"]
    frontend["Frontend React + Vite :5173"]
    backend["Backend NestJS :3000"]
    db[("PostgreSQL 16")]
    files[("backend/uploads")]
  end

  browser -->|"http://localhost"| nginx
  nginx -->|"/"| frontend
  nginx -->|"/api"| backend
  nginx -->|"/socket.io"| backend

  frontend -->|"HTTP JSON"| backend
  frontend -->|"WebSocket admin chatbot"| backend
  frontend -->|"iframe/enlace externo (consentido)"| maps

  backend -->|"Prisma"| db
  backend -->|"serveStatic /uploads"| files
  backend -->|"correo saliente"| smtp
```

## Estado actual

- Redis no existe en la topología actual.
- `frontend` y `backend` se ejecutan en modo desarrollo dentro de contenedor, pero el backend usa `yarn start`, no watch, para evitar procesos huérfanos.
- Nginx es el punto de entrada único esperado para pruebas funcionales.
