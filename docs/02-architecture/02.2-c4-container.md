# 02.2 - C4 Container

## Objetivo

Describir los contenedores tecnicos principales y sus relaciones.

```mermaid
flowchart LR
  user["Usuario (visitante, registrado o admin)"]
  google["Google Identity"]
  smtp["SMTP Provider"]

  subgraph system["Plataforma Proclade"]
    nginx["Nginx Reverse Proxy"]
    frontend["Frontend<br/>React + Vite"]
    api["Backend API<br/>NestJS"]
    db[("PostgreSQL")]
    redis[("Redis")]
  end

  user -->|"HTTP/HTTPS"| nginx
  nginx -->|"Proxy /"| frontend
  nginx -->|"Proxy /api + /socket.io"| api

  frontend -->|"JSON/HTTP"| api
  api -->|"Prisma + SQL"| db
  api -->|"TCP"| redis
  api -->|"HTTPS"| google
  api -->|"SMTP"| smtp
```

## Notas

- En desarrollo se ejecuta todo por Docker Compose.
- Nginx expone `localhost:80` y enruta a `frontend:5173` y `backend:3000`.
- Redis esta desplegado y disponible, aunque el uso funcional actual es limitado.
