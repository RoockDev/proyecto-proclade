# 02.2 - C4 Container

## Objetivo

Describir contenedores tecnicos principales y sus relaciones.

```mermaid
flowchart LR
  browser["Navegador web"]
  google["Google Identity"]
  smtp["SMTP Provider"]
  maps["Google Maps (iframe)"]

  subgraph system["Plataforma Proclade"]
    nginx["Nginx Reverse Proxy"]
    frontend["Frontend React + Vite"]
    api["Backend API NestJS"]
    db[("PostgreSQL")]
    redis[("Redis")]
    files[("Storage local /uploads")]
  end

  browser -->|"HTTP/HTTPS"| nginx
  nginx -->|"/"| frontend
  nginx -->|"/api"| api
  nginx -->|"/socket.io"| api

  frontend -->|"JSON HTTP"| api
  frontend -->|"iframe"| maps

  api -->|"Prisma"| db
  api -->|"cache/futuro"| redis
  api -->|"servir y guardar ficheros"| files
  api -->|"OAuth"| google
  api -->|"SMTP"| smtp
```

## Notas

- En local se ejecuta sobre `docker compose`.
- Nginx es el punto de entrada unico y aplica cabeceras de seguridad.
- `/uploads` se sirve desde backend y se usa para imagenes/PDFs administrables.
