# 02.4 - Flujos runtime clave

## Flujo A - Arranque limpio del proyecto en Docker

```mermaid
sequenceDiagram
  participant Dev as Operador
  participant Docker as Docker Compose
  participant DB as PostgreSQL
  participant API as Backend

  Dev->>Docker: docker compose up --build
  Docker->>DB: levantar servicio db
  Docker->>API: ejecutar backend/scripts/docker-start-dev.sh
  API->>API: prisma generate
  API->>DB: prisma migrate deploy
  API->>DB: prisma:init-system
  API->>DB: asegurar roles ADMIN/USER
  API->>DB: crear o verificar admin inicial
  API->>DB: cargar base inicial del chatbot
  API-->>Docker: backend listo
```

## Flujo B - Registro con privacidad y sesión

```mermaid
sequenceDiagram
  participant U as Usuario
  participant FE as Frontend
  participant API as AuthController/AuthService
  participant DB as PostgreSQL

  U->>FE: completa registro + acepta privacidad
  FE->>API: POST /auth/register
  API->>API: valida política de contraseña
  API->>DB: crea usuario + privacyAcceptedAt
  API->>DB: asegura rol USER
  API-->>FE: ApiResponse con accessToken y user
  FE->>FE: guarda accessToken y authUser en localStorage
```

## Flujo C - Mapa de delegación condicionado por cookies

```mermaid
sequenceDiagram
  participant U as Visitante
  participant FE as Frontend
  participant LS as localStorage
  participant Maps as Google Maps

  U->>FE: entra en Bibliotecas Humanas
  FE->>LS: leer puch_cookie_preferences_v1
  alt externalServices = true
    FE->>Maps: cargar iframe
    Maps-->>FE: mapa renderizado
  else externalServices = false
    FE-->>U: placeholder + opción aceptar o abrir enlace externo
  end
```

## Flujo D - Chatbot público + actualización en tiempo real del panel admin

```mermaid
sequenceDiagram
  participant U as Usuario web
  participant FE as Widget chatbot
  participant API as ChatbotController/Orchestrator
  participant DB as PostgreSQL
  participant WS as ChatbotAdminGateway
  participant Admin as Panel admin

  U->>FE: envía mensaje
  FE->>API: POST /chatbot/message
  API->>DB: guarda mensaje USER
  API->>API: matching + contexto dinámico + safety guard
  API->>DB: guarda respuesta BOT
  alt respuesta fallback
    API->>DB: registra unresolved question
  end
  API->>WS: emitir eventos admin
  WS-->>Admin: newMessage / unresolvedQuestionCreated
  API-->>FE: respuesta del chatbot
```
