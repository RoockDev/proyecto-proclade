# 02.4 - Flujos Runtime Clave

## Flujo A - Login + sesion valida

```mermaid
sequenceDiagram
  participant U as Usuario
  participant FE as Frontend
  participant API as AuthController/AuthService
  participant DB as PostgreSQL

  U->>FE: Envia login
  FE->>API: POST /auth/login
  API->>DB: Busca usuario activo
  DB-->>API: Usuario + roles
  API->>API: Valida password + genera JWT
  API-->>FE: ApiResponse<{accessToken,user}>
  FE->>FE: saveAuthSession()
  FE-->>U: Sesion iniciada
```

## Flujo B - Token expirado en frontend

```mermaid
sequenceDiagram
  participant FE as Frontend
  participant API as Backend

  FE->>FE: getAuthSession()
  FE->>FE: Lee exp del JWT
  alt Token expirado
    FE->>FE: clearAuthSession()
    FE-->>FE: Estado no autenticado
  else Token vigente
    FE->>API: Request protegida
    API-->>FE: 401
    FE->>FE: Axios interceptor limpia sesion
  end
```

## Flujo C - Bibliotecas Humanas publicas

```mermaid
sequenceDiagram
  participant V as Visitante
  participant FE as HumanLibrariesPage
  participant API as Regions/HumanBooks
  participant DB as PostgreSQL

  V->>FE: Abre /bibliotecas-humanas/:delegationSlug?
  FE->>API: GET /regions
  FE->>API: GET /human-books
  API->>DB: Consulta delegaciones y libros publicados
  DB-->>API: Datos activos
  API-->>FE: ApiResponse<List>
  FE-->>V: Tabs, contacto, mapa e items descargables
```

## Flujo D - Chatbot publico con contexto dinamico

```mermaid
sequenceDiagram
  participant U as Usuario
  participant FE as ChatbotWidget
  participant BOT as ChatbotController
  participant ORC as ChatbotOrchestrator
  participant DB as PostgreSQL

  U->>FE: Envia mensaje
  FE->>BOT: POST /chatbot/message
  BOT->>ORC: sendMessage(sessionId, pageContext, text)
  ORC->>DB: Lee knowledge + intent phrases + contexto dinamico
  ORC->>ORC: Matching (keyword,fuzzy,semantic,context)
  ORC-->>BOT: replyType + answer + suggestions + ctaLinks
  BOT-->>FE: ApiResponse<ChatbotReplyData>
  FE-->>U: Muestra respuesta y acciones sugeridas
```

## Flujo E - Gestion admin de superheroes (borrado definitivo)

```mermaid
sequenceDiagram
  participant A as Admin
  participant FE as AdminSuperheroesPage
  participant API as AdminSuperheroesController
  participant SVC as SuperheroesService
  participant DB as PostgreSQL

  A->>FE: Desactiva heroe
  FE->>API: PATCH /admin/superheroes/:id/status
  API->>SVC: updateStatus()
  SVC->>DB: Soft delete (deletedAt)

  A->>FE: Confirma eliminar definitivamente
  FE->>API: DELETE /admin/superheroes/:id/permanent
  API->>SVC: removePermanently()
  SVC->>DB: Elimina fila + limpia referencias de users
  API-->>FE: confirmacion
```
