# 02.1 - C4 Context

## Objetivo

Mostrar actores y sistemas externos que interactuan con la plataforma.

```mermaid
flowchart LR
  guest["Visitante"]
  member["Usuario registrado"]
  admin["Administrador ONG"]

  platform["Plataforma Proclade / Equipo PUCH<br/>Web fullstack de contenido, colaboracion y gestion"]

  google["Google Identity<br/>OAuth/OpenID"]
  smtp["SMTP Provider<br/>Correo de recuperacion"]
  maps["Google Maps<br/>Mapa embebido por iframe"]

  guest -->|"HTTPS"| platform
  member -->|"HTTPS + JWT"| platform
  admin -->|"HTTPS + JWT"| platform

  platform -->|"Verifica idToken"| google
  platform -->|"Envia reset password"| smtp
  platform -->|"Carga mapa embebido"| maps
```

## Notas

- El panel admin comparte frontend con la parte publica, pero con rutas y permisos separados.
- La seguridad real de acceso se aplica en backend, no solo en navegacion frontend.
- El chatbot forma parte del producto publico y tiene backoffice admin para gobernanza.
