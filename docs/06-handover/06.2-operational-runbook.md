# 06.2 - Runbook de operacion

## Incidencia: API no responde

1. Verificar contenedores:

```bash
docker ps
```

2. Revisar logs backend:

```bash
docker logs proyectoproclade-backend-1 --tail 200
```

3. Confirmar DB healthy (`pg_isready`) y `DATABASE_URL` correcta.

## Incidencia: frontend sin datos

1. Revisar `VITE_API_BASE_URL` / proxy Nginx `/api`.
2. Validar request real en devtools Network.
3. Revisar si backend responde 401 por token expirado.

## Incidencia: acceso admin inconsistente

1. Confirmar JWT vigente (frontend limpia sesion si expiro).
2. Validar roles del usuario (`ADMIN`) en backend.
3. Verificar `GET /users/me` y respuesta de guards en `/admin/*`.

## Incidencia: mapa o bibliotecas humanas no carga

1. Confirmar delegaciones activas en `/api/regions`.
2. Revisar CSP `frame-src` en `nginx/default.conf`.
3. Verificar que la URL de iframe se construye con slug valido.

## Incidencia: chatbot responde mal o no responde

1. Revisar `/api/chatbot/health`.
2. Confirmar seed de chatbot aplicada (`yarn prisma:seed`).
3. Revisar knowledge/intents desde `/admin/chatbot`.
4. Inspeccionar `UnresolvedQuestion` y metricas en backoffice.

## Incidencia: upload de imagen/PDF falla

1. Verificar permisos de `backend/uploads`.
2. Confirmar tamano y tipo de archivo permitido.
3. Revisar logs de controladores admin (`news`, `superheroes`, `human-books`).

## Comandos de recuperacion rapida

```bash
docker compose restart backend
docker compose restart frontend
docker compose restart nginx
```

Reset destructivo:

```bash
docker compose down -v
docker compose up --build
```
