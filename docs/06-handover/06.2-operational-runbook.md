# 06.2 - Runbook operativo

## Incidencia: la web no abre en `localhost`

1. Ejecutar:

```bash
docker compose ps
```

2. Verificar que `nginx` está arriba.
3. Revisar si el puerto `80` está ocupado por otra aplicación.

## Incidencia: el backend no responde

1. Revisar logs:

```bash
docker compose logs --tail=200 backend
```

2. Validar:

- `JWT_SECRET`
- `DATABASE_URL`
- que `db` esté healthy

## Incidencia: no se puede acceder al panel admin

1. Confirmar que existe sesión en frontend.
2. Confirmar que el usuario tiene rol `ADMIN`.
3. Si no se definieron `SYSTEM_ADMIN_*`, probar primero con:
   - `admin@equipo-puch.local`
   - `CambiaEsta123!`
4. Si el admin inicial se cambió en `.env` después de crear la base, recordar que eso no actualiza la contraseña del usuario ya existente.
5. Si hace falta regenerar admin inicial desde cero:

```bash
docker compose down -v
docker compose up --build
```

6. Antes de regenerar, revisar si `backend/uploads` contiene ficheros que deban conservarse, porque el reseteo de base no implica necesariamente limpiar ese directorio del host.

## Incidencia: registro o Colabora falla por privacidad

1. Revisar que el frontend envía `privacyAccepted: true`.
2. Revisar DTOs backend:
   - `RegisterDto`
   - `ContactFormDto`

## Incidencia: no se envían correos

1. Revisar `SMTP_HOST`, `SMTP_USER`, `SMTP_PASS`.
2. Si SMTP no está configurado:
   - `forgot-password` no enviará correo real
   - `colabora/contact` devolverá error

## Incidencia: el mapa no carga

1. Revisar preferencias `puch_cookie_preferences_v1`.
2. Asegurarse de que `externalServices = true`.
3. Si no se quiere aceptar, comprobar al menos que el enlace externo a Google Maps funciona.

## Incidencia: el chatbot responde mal

1. Revisar:

```bash
curl http://localhost/api/chatbot/health
```

2. Revisar configuración del motor en admin chatbot.
3. Revisar `UnresolvedQuestion` y knowledge items.

## Incidencia: el panel admin del chatbot no refleja tiempo real

1. Comprobar `socket.io` en Nginx.
2. Revisar conexión del frontend a `/admin/chatbot`.
3. Revisar logs del backend gateway.
