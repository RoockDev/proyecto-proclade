# 06.2 - Runbook de operacion

## Incidencia: API no responde

1. `docker ps` para validar contenedores activos.
2. Revisar logs backend:

```bash
docker logs proyectoproclade-backend-1 --tail 200
```

3. Verificar DB saludable (`pg_isready` ya en healthcheck).
4. Comprobar `DATABASE_URL` en backend.

## Incidencia: frontend sin datos

1. Revisar `VITE_API_BASE_URL`.
2. Confirmar proxy Nginx `/api/`.
3. Comprobar red en browser (devtools network).

## Incidencia: login falla en Google

1. Revisar `GOOGLE_CLIENT_ID` en backend y frontend.
2. Validar dominio/origen permitido en Google Cloud Console.
3. Revisar logs de `GoogleAuthService`.

## Incidencia: reset password no llega por email

1. Validar credenciales SMTP (`SMTP_*`).
2. Comprobar salida de logs en `MailService`.
3. Verificar que proveedor SMTP no bloquea envio.

## Comandos de recuperacion rapida

```bash
docker compose down
docker compose up --build
```

Reset completo (destructivo):

```bash
docker compose down -v
docker compose up --build
```
