# 06.1 - Guia de onboarding tecnico

## Objetivo

Permitir que un nuevo desarrollador levante el proyecto, valide el flujo principal y entienda la arquitectura en menos de una jornada.

## Paso 1 - Clonar

```bash
git clone <repo-url>
cd proyecto-proclade
```

## Paso 2 - Configurar entorno

1. Copiar `.env.example` -> `.env` (raiz).
2. Copiar `backend/.env.example` -> `backend/.env`.
3. Copiar `frontend/.env.example` -> `frontend/.env`.

## Paso 3 - Levantar stack

```bash
docker compose up --build
```

## Paso 4 - Validar funcionamiento base

1. Abrir `http://localhost`.
2. Probar `GET http://localhost/api/news`.
3. Probar login en UI y acceso a `/admin` con usuario admin seed.
4. Probar `GET http://localhost/api/chatbot/health`.

## Paso 5 - Lectura minima obligatoria

1. `docs/01-project-overview.md`
2. `docs/02-architecture/*`
3. `docs/03-backend/*`
4. `docs/04-frontend/*`
5. `docs/08-chatbot/*`
