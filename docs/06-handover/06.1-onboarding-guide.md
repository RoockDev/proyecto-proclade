# 06.1 - Onboarding técnico

## Objetivo

Permitir que un nuevo desarrollador o mantenedor levante el proyecto y valide el flujo principal en menos de una jornada.

## Paso 1 - Clonar

```bash
git clone <repo-url>
cd proyecto-proclade
```

## Paso 2 - Preparar configuración

1. Copiar `/.env.example` a `/.env`
2. Copiar `backend/.env.example` a `backend/.env`
3. Copiar `frontend/.env.example` a `frontend/.env`

## Paso 3 - Revisar mínimos obligatorios

- `JWT_SECRET` no vacío
- `DATABASE_URL` coherente con `/.env`
- `VITE_API_BASE_URL=/api`
- `SYSTEM_ADMIN_*` definido si se quiere controlar el admin inicial

## Paso 4 - Levantar

```bash
docker compose up --build
```

En el primer arranque limpio el backend:

1. aplica migraciones
2. inicializa roles `ADMIN` y `USER`
3. crea o verifica el administrador inicial
4. carga la base de conocimiento del chatbot

Si no se han definido `SYSTEM_ADMIN_*`, el acceso inicial por defecto será:

- email: `admin@equipo-puch.local`
- contraseña: `CambiaEsta123!`

Conviene cambiar esa contraseña tras el primer acceso o definir valores propios antes del primer arranque real.

## Paso 5 - Validar funcionamiento base

1. Abrir `http://localhost`
2. Verificar que cargan:
   - home
   - noticias
   - superhéroes
   - bibliotecas humanas
3. Entrar con el administrador inicial
4. Abrir `/admin`
5. Probar `GET http://localhost/api/chatbot/health`

Si el equipo cambia `SYSTEM_ADMIN_*` después de haber arrancado ya con una base existente, esas variables no reasignan automáticamente la contraseña del usuario que ya fue creado.
