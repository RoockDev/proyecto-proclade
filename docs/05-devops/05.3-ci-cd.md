# 05.3 - CI/CD en GitHub Actions

## Workflows actuales

### 1) CI Build (`.github/workflows/build.yml`)

Trigger:

- `pull_request` hacia `dev` y `main`.

Jobs:

1. `backend-build`
- instala dependencias backend con Yarn 4
- genera cliente Prisma
- build backend

2. `frontend-build`
- depende de `backend-build`
- instala dependencias frontend
- build frontend

### 2) Deploy to Staging (`.github/workflows/deploy-staging.yml`)

Triggers:

- `push` en `dev` (paths relevantes)
- `pull_request` hacia `dev` (validacion)
- `workflow_dispatch`

Flujo actual:

1. `lint` (placeholder)
2. `build` (placeholder)
3. `test` (placeholder)
4. `deploy` (valida secretos + prueba conectividad SSH)

## Estado de madurez

- CI de build funcional.
- Pipeline de deploy con base tecnica lista, pero con jobs de calidad aun pendientes de cierre.

## Recomendaciones inmediatas

1. Sustituir placeholders por comandos reales (`yarn lint`, `yarn test`, `yarn build`).
2. Añadir healthcheck post-deploy.
3. Registrar version desplegada por release/tag o SHA visible.
