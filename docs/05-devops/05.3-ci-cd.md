# 05.3 - CI/CD en GitHub Actions

## Workflows actuales

### 1. CI Build

Archivo: `.github/workflows/build.yml`

Trigger:

- Pull request contra `dev` y `main`.

Jobs:

1. `backend-build`
- instala dependencias backend
- genera cliente Prisma
- build backend

2. `frontend-build`
- depende de backend-build
- instala dependencias frontend
- build frontend

### 2. Deploy to Staging

Archivo: `.github/workflows/deploy-staging.yml`

Trigger:

- `push` a `dev` (rutas relevantes)
- `pull_request` a `dev` (solo validacion)
- `workflow_dispatch`

Flujo actual:

1. `lint` (placeholder)
2. `build` (placeholder)
3. `test` (placeholder)
4. `deploy` (valida secretos y SSH connectivity)

## Estado de madurez actual

- CI de build funcional.
- Deploy staging base montado pero no completo end-to-end.
- Lint/test en pipeline de deploy pendientes de cierre definitivo.

## Recomendaciones inmediatas

1. Sustituir placeholders por comandos reales (`yarn lint`, `yarn test`).
2. AĂ±adir healthcheck post-deploy.
3. Publicar version/tag de release en cada despliegue estable.
