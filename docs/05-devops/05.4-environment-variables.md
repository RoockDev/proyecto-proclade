# 05.4 - Variables de entorno

## Archivo raiz `.env`

Usado principalmente por Docker Compose para servicios base.

- `POSTGRES_USER`
- `POSTGRES_PASSWORD`
- `POSTGRES_DB`
- `GOOGLE_CLIENT_ID`

## Backend (`backend/.env`)

Referencia: `backend/.env.example`

- `DATABASE_URL`
- `REDIS_URL`
- `CORS_ORIGIN`
- `JWT_SECRET`
- `JWT_EXPIRES_IN`
- `GOOGLE_CLIENT_ID`
- `SMTP_HOST`
- `SMTP_PORT`
- `SMTP_USER`
- `SMTP_PASS`
- `MAIL_FROM`
- `FRONTEND_URL`
- `RESET_PASSWORD_TTL_MINUTES`

## Frontend (`frontend/.env`)

Referencia: `frontend/.env.example`

- `VITE_API_URL`
- `VITE_API_BASE_URL`
- `VITE_SOCKET_PATH`
- `VITE_GOOGLE_CLIENT_ID`
- `VITE_NEWS_FORCE_MOCKS`

## Politica de secretos

1. No commitear `.env` reales.
2. Mantener siempre `.env.example` actualizado.
3. Para CI/CD usar GitHub Secrets.
