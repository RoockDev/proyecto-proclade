# 05.4 - Variables de entorno

## Raíz del proyecto (`/.env`)

Usado por Docker Compose:

- `POSTGRES_USER`
- `POSTGRES_PASSWORD`
- `POSTGRES_DB`

## Backend (`backend/.env`)

Referencia:

- `backend/.env.example`

### Base

- `DATABASE_URL`
- `CORS_ORIGIN`
- `JWT_SECRET`
- `JWT_EXPIRES_IN`

### Admin inicial

- `SYSTEM_ADMIN_EMAIL`
- `SYSTEM_ADMIN_PASSWORD`
- `SYSTEM_ADMIN_NAME`
- `SYSTEM_ADMIN_SURNAME`

Notas:

- valores por defecto del bootstrap si no se definen:
  - `SYSTEM_ADMIN_EMAIL=admin@equipo-puch.local`
  - `SYSTEM_ADMIN_PASSWORD=CambiaEsta123!`
  - `SYSTEM_ADMIN_NAME=Administrador`
  - `SYSTEM_ADMIN_SURNAME=Equipo PUCH`
- `SYSTEM_ADMIN_PASSWORD` debe cumplir la política de contraseña
- el admin inicial se crea o verifica en el primer arranque limpio de base de datos
- si el admin ya existe en base de datos, cambiar esta variable no le cambia la contraseña automáticamente
- si se quiere entregar el sistema con otras credenciales iniciales, hay que definir `SYSTEM_ADMIN_*` antes del primer `docker compose up --build` sobre una base vacía

### SMTP y formularios

- `SMTP_HOST`
- `SMTP_PORT`

- `MAIL_FROM`
- `CONTACT_FORM_TO`

### Recuperación de contraseña

- `FRONTEND_URL`
- `RESET_PASSWORD_TTL_MINUTES`

### Chatbot

- `CHATBOT_WEIGHT_KEYWORD`
- `CHATBOT_WEIGHT_FUZZY`
- `CHATBOT_WEIGHT_SEMANTIC`
- `CHATBOT_WEIGHT_CONTEXT`
- `CHATBOT_THRESHOLD_DIRECT_ANSWER`
- `CHATBOT_THRESHOLD_CLARIFICATION`
- `CHATBOT_FUZZY_INTERNAL_MIN`

## Frontend (`frontend/.env`)

Referencia:

- `frontend/.env.example`

Variables activas:

- `VITE_API_BASE_URL`
- `VITE_SOCKET_PATH`

## Reglas

1. No commitear `.env` reales.
2. Mantener `.env.example` sincronizados con el código.
3. Si se cambia `.env` raíz, revisar también `DATABASE_URL`.
4. Para una instalación limpia entregable, conviene definir `SYSTEM_ADMIN_*` antes del primer arranque real.
