# 05.4 - Variables de entorno

## Archivo raiz `.env`

Usado por Docker Compose para servicios base:

- `POSTGRES_USER`
- `POSTGRES_PASSWORD`
- `POSTGRES_DB`
- `GOOGLE_CLIENT_ID`

## Backend (`backend/.env`)

Referencia: `backend/.env.example`

Base:
- `DATABASE_URL`
- `REDIS_URL`
- `CORS_ORIGIN`
- `JWT_SECRET`
- `JWT_EXPIRES_IN`

Auth/Email:
- `GOOGLE_CLIENT_ID`
- `SMTP_HOST`
- `SMTP_PORT`
- `SMTP_USER`
- `SMTP_PASS`
- `MAIL_FROM`
- `FRONTEND_URL`
- `RESET_PASSWORD_TTL_MINUTES`

Chatbot matching:
- `CHATBOT_WEIGHT_KEYWORD`
- `CHATBOT_WEIGHT_FUZZY`
- `CHATBOT_WEIGHT_SEMANTIC`
- `CHATBOT_WEIGHT_CONTEXT`
- `CHATBOT_THRESHOLD_DIRECT_ANSWER`
- `CHATBOT_THRESHOLD_CLARIFICATION`
- `CHATBOT_FUZZY_INTERNAL_MIN`

## Frontend (`frontend/.env`)

Referencia: `frontend/.env.example`

- `VITE_API_URL`
- `VITE_API_BASE_URL`
- `VITE_SOCKET_PATH`
- `VITE_GOOGLE_CLIENT_ID`
- `VITE_NEWS_FORCE_MOCKS`

## Politica de secretos

1. Nunca commitear `.env` reales.
2. Mantener `.env.example` sincronizado con el codigo.
3. En CI/CD usar GitHub Secrets por entorno.
