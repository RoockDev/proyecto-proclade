# 05.1 - Entorno local con Docker

## Servicios definidos

`docker-compose.yml` levanta:

1. `db`
2. `backend`
3. `frontend`
4. `nginx`

Redis ya no forma parte del entorno.

## Flujo de arranque recomendado

```bash
docker compose up --build
```

## Qué hace el arranque

- construye imágenes
- levanta PostgreSQL
- ejecuta migraciones
- ejecuta bootstrap del sistema
- deja la app accesible en `http://localhost`

## Volúmenes y mounts

- `db_data`
- `backend_node_modules`
- `frontend_node_modules`
- bind mount `./backend/uploads:/app/uploads`

## Reset completo

```bash
docker compose down -v
docker compose up --build
```

## Importante sobre `uploads`

`docker compose down -v` limpia los volúmenes Docker, pero no garantiza limpiar `backend/uploads` porque es un bind mount del host.

## Compatibilidad de scripts

Existe `.gitattributes` para forzar finales de línea LF en scripts sensibles, especialmente:

- `backend/scripts/docker-start-dev.sh`

Esto evita roturas en Windows al ejecutar `/bin/sh` dentro del contenedor.
