# 05.2 - Red, Nginx y exposición

## Punto de entrada

Archivo:

- `nginx/default.conf`

Nginx expone `:80` y enruta:

- `/` -> `frontend:5173`
- `/api/` -> `backend:3000/`
- `/socket.io/` -> `backend:3000/socket.io/`
- `/.vite/` -> frontend
- `/node_modules/.vite/` -> frontend

## Seguridad HTTP aplicada

Cabeceras activas:

- `X-Content-Type-Options`
- `X-Frame-Options`
- `Referrer-Policy`
- `Permissions-Policy`
- `Content-Security-Policy`

## CSP actual

Puntos relevantes:

- `script-src 'self' 'unsafe-inline'`
- `style-src 'self' 'unsafe-inline' https://fonts.googleapis.com`
- `img-src 'self' data: https://*.gstatic.com`
- `font-src 'self' https://fonts.gstatic.com`
- `connect-src 'self'`
- `frame-src 'self' https://www.google.com https://maps.google.com`

Interpretación:

- la app permite técnicamente Google Fonts y Google Maps
- funcionalmente el mapa se gatea desde frontend por preferencias del usuario
