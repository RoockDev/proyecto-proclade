# 05.2 - Topologia de red y Nginx

## Rol de Nginx

Archivo: `nginx/default.conf`.

Nginx escucha en `:80` y enruta:

- `/` -> `frontend:5173`
- `/api/` -> `backend:3000/`
- `/socket.io/` -> `backend:3000/socket.io/`
- `/.vite/` y `/node_modules/.vite/` -> frontend dev server

## Seguridad HTTP aplicada

Cabeceras activas con `always`:

- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: SAMEORIGIN`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Permissions-Policy: geolocation=(), microphone=(), ...`
- `Content-Security-Policy` restrictiva

Puntos CSP relevantes:

- `connect-src 'self'` para API/socket sobre mismo host.
- `frame-src 'self' https://www.google.com https://maps.google.com` para mapa embebido.

## Bloqueos de rutas sensibles

- Bloqueo de `.env`, `.git`, `.ht*`, archivos ocultos y backups (`.bak`, `.old`, `~`).
- Excepcion controlada para `/.well-known/`.

## Recomendaciones para produccion

1. TLS completo (443) + redirect HTTP->HTTPS.
2. Rate limiting en rutas sensibles (`/auth/*`, `/chatbot/*`, `/admin/*`).
3. Revisar y endurecer CSP en cada nueva integracion externa.
