# 07.1 - Estrategia de testing

## Estado real actual

### Backend

- suite de tests unitarios y de servicio en crecimiento
- cobertura útil especialmente en:
  - auth
  - users
  - mail
  - news
  - superheroes
  - chatbot

### Frontend

- sin suite automatizada consolidada
- validación principal basada en:
  - `yarn lint`
  - `yarn build`
  - smoke manual funcional

### CI

- build backend y frontend automatizados
- staging aún sin lint/build/test reales

## Estrategia recomendada para continuidad

1. Mantener tests de backend en servicios con lógica de negocio.
2. Priorizar frontend tests para:
   - auth session storage
   - guards de admin
   - formularios públicos con privacidad
   - chatbot widget
   - admin chatbot realtime
3. Añadir smoke automatizado de rutas públicas y admin.

## Definition of Done técnica recomendada

Una HU o refactor debería considerarse cerrada cuando:

1. respeta el contrato HTTP
2. no rompe Docker startup
3. pasa build/lint/test relevantes
4. deja la documentación técnica actualizada
