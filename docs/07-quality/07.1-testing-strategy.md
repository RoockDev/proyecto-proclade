# 07.1 - Estrategia de testing

## Estado actual

- Backend:
  - unit tests existentes en `news.service.spec.ts`
  - e2e para noticias en `backend/test/news/news.e2e-spec.ts`
- Frontend:
  - sin suite automatizada consolidada en esta version
- CI:
  - workflow de build activo
  - lint/test de deploy staging aun en construccion por HUs

## Estrategia recomendada (siguiente fase)

1. Backend
- Mantener unit tests por servicio con reglas de negocio.
- Aumentar e2e para auth y users.

2. Frontend
- Añadir tests de componentes y rutas criticas.
- Validar estados `loading/error/empty` en noticias.

3. Integracion
- Añadir smoke tests de rutas publicas tras deploy staging.

## Definition of Done tecnica sugerida

Una HU no se considera completa si no cumple:

1. Contrato API respetado.
2. Lint y build en verde.
3. Cobertura de pruebas para camino feliz y error principal.
4. Documentacion tecnica actualizada en `docs/`.
