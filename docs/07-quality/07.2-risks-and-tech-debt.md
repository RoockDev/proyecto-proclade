# 07.2 - Riesgos tecnicos y deuda

## Riesgos prioritarios (P1)

1. Pipeline de staging incompleto
- Impacto: regresiones no detectadas antes de deploy.
- Mitigacion: cerrar jobs reales de lint/test en `deploy-staging.yml`.

2. Guard frontend admin pendiente
- Impacto: UX confusa (acceso visual a rutas sin sesion).
- Mitigacion: añadir route guard en router frontend.

3. Fallback de secreto JWT
- Impacto: riesgo seguridad si falta env en despliegue.
- Mitigacion: forzar variable obligatoria sin default inseguro.

## Riesgos medios (P2)

1. Falta observabilidad estructurada
- Impacto: diagnostico lento en incidencias reales.
- Mitigacion: logging estructurado + trazas + alertas basicas.

2. Redis sin estrategia funcional clara
- Impacto: coste operativo sin retorno inmediato.
- Mitigacion: definir casos (cache, colas, rate limit) o retirar temporalmente.

3. Paginacion de noticias en cliente
- Impacto: escalabilidad limitada si crece mucho el volumen.
- Mitigacion: exponer paginacion server-side con metadata.

## Riesgos bajos (P3)

1. Documentacion funcional no versionada por release.
2. Falta ADR formal por cada cambio arquitectonico importante.

## Plan sugerido de reduccion de deuda

1. Sprint 1: CI real + hardening auth.
2. Sprint 2: observabilidad + guard frontend.
3. Sprint 3: paginacion backend + limpieza Redis strategy.
