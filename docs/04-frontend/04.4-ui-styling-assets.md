# 04.4 - UI, estilos y assets

## Sistema visual

Fuentes principales:

- `frontend/src/styles/globals.css`
- `frontend/src/styles/bootstrap-overrides.css`
- CSS por feature/componente
- Guia de estilos PDF: `frontend/documentación/GuíaDeEstilosProclade.pdf`

## Criterios de estilo usados

1. Bootstrap para layout base y componentes comunes.
2. Clases propias para personalizacion de marca.
3. Estilos por seccion/feature para reducir acoplamiento.

## Assets relevantes

- Logo marca: `frontend/src/assets/branding/logo-equipo-puch.jpg`
- Hero y recursos home: `frontend/src/assets/home/*`
- Superheroes: `frontend/src/assets/superheroes/*`
- Imagenes proyectos campo: `frontend/src/assets/Fotos proyectos equipo puch/*`
- Paleta entregada cliente: `frontend/src/assets/paleta de colores.jpeg`

## Recomendaciones para continuidad

1. Evitar sobrescrituras destructivas de clases base Bootstrap.
2. Mantener variables de color en hojas globales.
3. Al agregar una nueva feature, crear carpeta propia con CSS local.
