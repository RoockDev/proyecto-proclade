# 04.4 - UI, estilos y assets

## Sistema visual

El proyecto mezcla:

- Bootstrap 5 para layout y utilidades base
- CSS por feature/componente
- clases compartidas de branding en estilos globales

## Patrones visuales vigentes

1. `page-hero` y `gradient-hero` para cabeceras de secciones públicas.
2. Cards modulares para noticias, superhéroes, retos y delegaciones.
3. Uso extensivo de `bootstrap-icons`.
4. Formularios públicos y admin con feedback inline.
5. Modales específicos para detalle de retos y superhéroes.

## Assets

- branding y logotipos: `frontend/src/assets/branding`
- imágenes home/proyectos: `frontend/src/assets/home`
- superhéroes base: `frontend/src/assets/superheroes`
- uploads dinámicos:
  - noticias
  - superhéroes
  - PDFs de bibliotecas humanas

## Reglas prácticas de continuidad

1. Si una feature pública consume datos remotos, debe contemplar `loading`, `empty` y `error`.
2. Si se añaden enlaces externos, deben abrir en nueva pestaña con `rel="noreferrer"`.
3. Si una vista usa un servicio externo, debe respetar las preferencias de cookies cuando aplique.
4. Si se añaden nuevas páginas públicas, mantener coherencia con hero, footer y header actuales.
