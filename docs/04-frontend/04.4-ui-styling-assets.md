# 04.4 - UI, estilos y assets

## Sistema visual

Fuentes principales:

- `frontend/src/styles/globals.css`
- `frontend/src/styles/bootstrap-overrides.css`
- CSS por feature/componente
- Guia de estilos PDF: `frontend/documentacion/GuiaDeEstilosProclade.pdf`

## Criterios de estilo aplicados

1. Bootstrap para layout y base utilitaria.
2. Clases propias para branding y componentes de negocio.
3. Arquitectura CSS por feature para minimizar acoplamiento.
4. Responsive first en navbar, secciones publicas y panel admin.

## Assets relevantes

- Favicon corporativo: `frontend/public/favicon-puch.png`
- Branding/logo: `frontend/src/assets/branding/*`
- Home/hero/proyectos: `frontend/src/assets/home/*` y `frontend/src/assets/Fotos proyectos equipo puch/*`
- Superheroes: `frontend/src/assets/superheroes/*`

## Reglas practicas para continuidad

1. No sobrescribir clases base globales de Bootstrap de forma destructiva.
2. Reutilizar componentes compartidos admin (`AdminButton`, `AdminDataTable`, `ConfirmModal`, badges).
3. Mantener truncado y accesibilidad en campos de upload con nombres largos.
4. Validar siempre estados `loading/empty/error` en pantallas con datos remotos.
