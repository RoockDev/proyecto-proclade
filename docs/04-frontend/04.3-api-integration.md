# 04.3 - Integración API y estado cliente

## Cliente HTTP

Archivo:

- `frontend/src/services/http/axios.instance.ts`

Base URL:

- `VITE_API_BASE_URL`
- fallback: `http://localhost:3000`

Comportamiento global:

- añade `Content-Type: application/json`
- limpia sesión si recibe `401`

## Sesión del usuario

Archivo:

- `frontend/src/features/auth/utils/auth-session.storage.ts`

Claves usadas en `localStorage`:

- `accessToken`
- `authUser`

Comportamiento:

1. guarda token y usuario tras login/registro
2. parsea el JWT localmente para comprobar expiración
3. borra sesión si el token ya expiró
4. emite evento propio para refrescar UI en la misma pestaña

## Preferencias de cookies

Archivo:

- `frontend/src/features/privacy/utils/cookie-preferences.ts`

Clave usada en `localStorage`:

- `puch_cookie_preferences_v1`

Categorías:

- `necessary`
- `externalServices`
- `analytics`

Estado actual:

- `necessary` siempre activa
- `externalServices` controla el mapa embebido
- `analytics` existe como categoría pero no hay analítica activa
