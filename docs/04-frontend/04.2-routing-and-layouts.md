# 04.2 - Routing y Layouts

Fuente: `frontend/src/router/app.router.tsx`.

## Rutas publicas (PublicLayout)

- `/` -> `HomePage`
- `/auth/login` -> `AuthPage` (login)
- `/auth/register` -> `AuthPage` (register)
- `/auth/reset-password` -> `ResetPasswordPage`
- `/superheroes` -> `SuperheroesPage` (lazy loaded con `Suspense`)
- `/noticias` -> `NewsListPage`
- `/noticias/:slug` -> `NewsDetailPage`
- `/bibliotecas-humanas` -> `HumanLibrariesPage`
- `/bibliotecas-humanas/:delegationSlug` -> `HumanLibrariesPage`
- `/colabora` -> `ColaboraPage`
- `*` -> `NotFoundPage`

## Rutas admin (AdminLayout)

- `/admin` -> `AdminDashboardPage`
- `/admin/noticias` -> `AdminNewsPage`
- `/admin/retos` -> `AdminChallengesPage`
- `/admin/libros` -> `AdminHumanBooksPage`
- `/admin/superheroes` -> `AdminSuperheroesPage`
- `/admin/delegaciones` -> `AdminRegionsPage`
- `/admin/usuarios` -> `AdminUsersPage`
- `/admin/chatbot` -> `AdminChatbotPage`
- `*` -> `NotFoundPage`

## Control de acceso frontend

`AdminLayout` aplica guard de UX:

1. Si no hay sesion valida -> redirige a `/auth/login?redirectTo=...`.
2. Si hay sesion pero no rol admin -> muestra vista de acceso restringido.

Nota: la seguridad efectiva sigue estando en backend (`JwtAuthGuard` + `RolesGuard`).
