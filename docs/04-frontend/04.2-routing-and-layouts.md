# 04.2 - Routing y Layouts

Fuente: `frontend/src/router/app.router.tsx`.

## Rutas publicas

- `/` -> HomePage
- `/auth/login` -> AuthPage (modo login)
- `/auth/register` -> AuthPage (modo register)
- `/auth/reset-password` -> ResetPasswordPage
- `/superheroes` -> SuperheroesPage
- `/noticias` -> NewsListPage
- `/noticias/:slug` -> NewsDetailPage
- `/colabora` -> ColaboraPage
- `*` -> NotFoundPage

Todas bajo `PublicLayout`.

## Rutas admin

- `/admin` -> AdminPanelPage
- `/admin/campanas` -> AdminCampaignsPage

Bajo `AdminLayout`.

## Observaciones

1. Enrutado es client-side con `createBrowserRouter`.
2. Actualmente no hay guard de ruta admin en frontend; la proteccion fuerte esta en backend por rol.
3. Se recomienda agregar route guard frontend para mejorar UX (no seguridad).
