# 04.1 - Vision Frontend

## Stack

- React 19
- TypeScript
- Vite 7
- React Router
- Axios
- Bootstrap 5 + CSS por feature

## Estructura por features

Carpetas principales:

- `features/home`: landing publica modular.
- `features/news`: listado/detalle de noticias.
- `features/superheroes`: listado paginado de superheroes.
- `features/human-libraries`: delegaciones + libros humanos.
- `features/auth`: login, registro, reset, utilidades de sesion.
- `features/profile`: modal de perfil y cambio de contrasena.
- `features/chatbot`: widget, API, estado y FAQ inicial.
- `features/admin`: dashboard y CRUD de dominio (news, retos, libros, regiones, usuarios, superheroes, chatbot).
- `router`: layouts y rutas.
- `services/http`: Axios base + interceptor global.

## Layouts

- Publico: `PublicLayout` con Header, Outlet, Footer y ChatbotWidget.
- Admin: `AdminLayout` con sidebar/topbar y control de acceso por sesion/rol.

## Principios de implementacion

1. Tipado estricto de contratos API.
2. Separacion UI/logica (componentes, hooks, api, mapper).
3. Estado local por feature; sin store global monolitico.
4. Consistencia visual via tokens globales + CSS scoped por componente.
