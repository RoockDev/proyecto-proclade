# 04.5 - Operación frontend

## Camino recomendado

La forma normal de operar la UI es levantar toda la stack:

```bash
docker compose up --build
```

## Comandos útiles para desarrollo local

```bash
cd frontend
corepack enable
corepack prepare yarn@4.12.0 --activate
yarn install
yarn lint
yarn build
```

`yarn dev` existe, pero el flujo principal del proyecto sigue siendo Docker + Nginx.

## Variables relevantes

Referencia:

- `frontend/.env.example`

Variables activas:

- `VITE_API_BASE_URL=/api`
- `VITE_SOCKET_PATH=`

## Smoke test mínimo recomendado

1. home pública carga en `http://localhost`
2. login y registro funcionan
3. el botón de panel aparece solo para admin
4. noticias y superhéroes cargan desde API real
5. bibliotecas humanas cargan delegaciones y libros
6. el mapa queda bloqueado si no se aceptan servicios externos
7. Colabora valida privacidad y maneja éxito/error
8. chatbot responde y muestra aviso de privacidad
9. panel admin del chatbot recibe actividad en tiempo real
